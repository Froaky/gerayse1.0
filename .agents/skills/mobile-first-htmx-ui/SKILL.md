description: build responsive operator-facing Django templates with HTMX or light JavaScript. use when creating forms, lists, dashboards, validation flows, and transactional screens that must work primarily on phones and also on desktop.
---

Design for real operational use on mobile first.

Always:
1. Start from the mobile workflow before considering desktop layout.
2. Keep critical actions visible, touch-friendly, and low-friction.
3. Use HTMX partials for fast updates, inline validation, filtered lists, and stepwise flows when it improves the experience.
4. Prefer cards, stacked sections, sticky summaries, and clear totals on mobile instead of dense tables by default.
5. Make errors, warnings, and validation failures obvious and actionable.
6. Preserve accessibility, readable spacing, and strong feedback for loading and success states.

Output format:
- Goal
- User workflow
- Screens or partials
- HTMX or JS interactions
- Responsive rules
- QA checklist
